# Pydantic 校验流程详解

> `model_validate` 和 `@field_validator` / `@model_validator` 各司其职

---

## 一、校验管线总览

Pydantic 的校验是一条线性管线，`model_validate()` 是启动按钮，`@field_validator` 和 `@model_validator` 是管线上的工位：

```
输入数据（dict / JSON 字符串）
    │
    ▼
pydantic-core (Rust 引擎)
  1. JSON 解析（如果是字符串）
  2. 类型转换（"123" → 123）
  3. 类型检查（值是否符合注解类型）
  失败 → ValidationError（含路径）
    │
    ▼  pydantic-core 校验通过
@field_validator（after 模式）
  逐个字段执行自定义校验
  每个字段可独立抛 ValueError
  可修改字段的值
    │
    ▼  所有 field_validator 通过
@model_validator（after 模式）
  跨字段逻辑校验
  可访问所有字段的值
  必须返回 self
    │
    ▼  全部通过
输出 Pydantic 实例
User(name="Alice", age=30)
```

关键顺序约束：
- **短路**：pydantic-core 的类型转换失败 → 不会进入 field_validator
- **顺序**：field_validator 按定义顺序执行 → model_validator
- **合并错误**：所有校验完成后一次性抛出 ValidationError，包含所有字段的错误

## 二、各自的职责

### model_validate — 整条管线的启动按钮

```python
user = User.model_validate({"name": "Alice", "age": "30"})
# 1. pydantic-core 启动 → "30" → 30（宽松模式自动转换）
# 2. 走 @field_validator（如果有）
# 3. 走 @model_validator（如果有）
# 4. 返回 User(name="Alice", age=30)
```

它是触发完整校验管线的入口，不只是"字典转模型"的快捷方式。即使传入正确类型的值，仍然会过完整管线。

两个重载：

| 方法 | 输入类型 | 用途 |
|------|---------|------|
| `model_validate(dict)` | Python 对象 | 直接传入字典 |
| `model_validate_json(str)` | JSON 字符串 | 先 JSON.parse 再过管线 |

### @field_validator — 单字段工位

```python
@field_validator("name")
@classmethod
def name_must_not_be_blank(cls, v: str) -> str:
    """只看 name 字段"""
    if v.strip() == "":
        raise ValueError("name cannot be blank")
    return v.strip()
```

- **视角**：单字段，看不到其他字段
- **时机**：pydantic-core 完成类型转换之后
- **职责**：字段级别的业务规则校验 + 数据清洗
- **可修改**：可修改并返回新值替代原始值

### @model_validator — 跨字段工位

```python
@model_validator(mode="after")
def check_dates(self) -> "Booking":
    """能看到所有字段"""
    if self.end <= self.start:
        raise ValueError("end must be after start")
    return self
```

- **视角**：全字段，可比较 start 和 end
- **时机**：所有 field_validator 通过之后
- **职责**：跨字段约束检查（如 start < end、password == confirm_password）
- **不可修改**：mode="after" 下一般只校验不修改；mode="before" 可预处理原始 dict

## 三、直觉类比

```
model_validate      → 整条流水线的启动开关
pydantic-core       → 自动质检机（类型、格式）
@field_validator    → 每个零件工位的专项质检员
@model_validator    → 总装完成后的整机测试员
```

---

## 四、常见误解

| 误解 | 真相 |
|------|------|
| 直接传 dict 给 `User(**data)` 不走校验 | 走。`__init__` 内部调用了 `model_validate` |
| `model_validate` 只做类型转换 | 做完整管线：转换 + 约束 + 自定义校验 |
| `@field_validator` 可以访问其他字段 | 不能。需要跨字段用 `@model_validator` |
| 校验失败返回不完整的实例 | 不返回。任何阶段失败都抛 `ValidationError` |
