# 查询重构与分发

> 来源：DataWhale《All-in-RAG》第四章第四节
> 核心目标：将用户的原始问题预处理成更适合检索的形式，并分发到正确数据源。

---

## 一、查询翻译（Query Translation）

弥合用户"自然语言提问"与文档库"存储信息"之间的语义鸿沟。四种主流技术：

### 1.1 提示工程（Prompt Engineering）

直接用 LLM 改写查询。进阶用法：让 LLM 输出**结构化指令**（JSON），代码执行排序/过滤逻辑。

**示例**：对"时间最短的视频"这类不可直接用 `SelfQueryRetriever` 处理的排序查询，让 LLM 输出 `{"sort_by": "length", "order": "asc"}`，再由代码执行排序。

### 1.2 多查询分解（Multi-query）

将复杂问题拆成多个子问题，分别检索后合并去重。

**示例**：
- 原始："《流浪地球》中刘慈欣对 AI 和未来社会有何看法？"
- 分解：
  - "《流浪地球》中描述的 AI 技术有哪些？"
  - "《流浪地球》中描绘的未来社会是怎样的？"
  - "刘慈欣关于 AI 的观点是什么？"

LangChain 提供了 `MultiQueryRetriever` 实现此流程。

### 1.3 退步提示（Step-Back Prompting）

Google DeepMind 提出。先"退一步"生成高层抽象问题，获得通用原理后再推理具体问题。

**示例**：
- 原始："理想气体温度增 2 倍、体积增 8 倍，压力如何变化？"
- 退步问题："背后的物理原理是什么？" → 理想气体定律 PV=nRT
- 推理：代入计算得压力变为 1/4

### 1.4 假设性文档嵌入（HyDE）

解决查询短 vs 文档长的语义鸿沟。三步流程：

1. **生成**：LLM 根据查询生成一个假设性的理想答案文档
2. **编码**：将假设文档向量化
3. **检索**：用该向量在数据库中找最相似的真实文档

将"查询→文档"匹配转化为"文档→文档"匹配。

## 二、查询路由（Query Routing）

智能调度中心，根据查询语义动态选择处理路径。

### 2.1 应用场景

| 场景 | 说明 | 示例 |
|------|------|------|
| **数据源路由** | 按意图分到不同知识库 | 产品文档 → 向量库 / 用户历史 → SQL |
| **组件路由** | 按复杂度选不同处理器 | 简单 FAQ → 向量检索 / 复杂操作 → Agent |
| **提示模板路由** | 按任务类型选提示模板 | 数学 → Step-by-Step / 代码 → 代码模板 |

### 2.2 实现方法

#### 2.2.1 基于 LLM 的意图识别（灵活）

三步：定义分类器 → 定义路由分支（`RunnableBranch`）→ 组合完整链。

```python
classifier_chain = classifier_prompt | llm | StrOutputParser()
router_branch = RunnableBranch(
    (lambda x: "川菜" in x["topic"], sichuan_chain),
    (lambda x: "粤菜" in x["topic"], cantonese_chain),
    general_chain  # default
)
full_chain = {"topic": classifier_chain, "question": lambda x: x["question"]} | router_branch
```

#### 2.2.2 嵌入相似性路由（低延迟）

用向量相似度替代 LLM 分类。流程：

1. 为每个路由写一段描述文本 → 向量化
2. 用户查询 → 向量化 → 计算余弦相似度
3. 选最相似的路由执行

### 2.3 LlamaIndex 拓展

- `RouterQueryEngine` + `QueryEngineTool`：每个 Tool 包含引擎 + 描述文本
- 用 `Selector`（`LLMSingleSelector` / `PydanticSingleSelector`）做语义匹配
- 无独立的嵌入路由组件，语义路由融合在 LLM 意图识别中

## 关键对比

| 技术 | 核心思路 | 适用场景 |
|------|---------|---------|
| 提示工程 | LLM 输出结构化指令 | 排序、过滤等可编程操作 |
| Multi-query | 分解 → 独立检索 → 合并 | 多维度复杂问题 |
| Step-Back | 抽象化 → 获取原理 → 推理 | 需要先验知识的推理题 |
| HyDE | 查询 → 假想文档 → 文档检索 | 查询短语与文档差异大 |
| 路由 | 分类/相似度 → 分发 | 多数据源/多处理器系统 |
