# 格式化生成

> 来源：DataWhale《All-in-RAG》第五章第一节
> 核心目标：让 LLM 输出结构化数据（JSON/XML），连接自然语言理解与程序化逻辑。

---

## 一、为什么需要格式化生成？

| 场景 | 需求 |
|------|------|
| 电商客服推荐键盘 | 返回 JSON 列表（名称、价格、特性、链接）→ 前端直接渲染 |
| 自然语言转 API | `"明天上海到北京的航班"` → `{"departure":"上海","destination":"北京","date":"2025-07-18"}` |
| 数据自动提取 | 从新闻中抽事件、时间、地点、人物 → 结构化入库 |

---

## 二、实现方法

### 2.1 LangChain Output Parsers

`OutputParsers` 在 prompt 中自动注入格式指令，并将 LLM 返回的纯文本解析为结构化对象。

常用解析器：

| 解析器 | 功能 |
|--------|------|
| `StrOutputParser` | 返回字符串 |
| `JsonOutputParser` | 解析复杂 JSON |
| `PydanticOutputParser` | 最严格：Pydantic 模型定义 + 验证 |

**PydanticOutputParser 流程**：

```python
# 1. 定义数据模型
class PersonInfo(BaseModel):
    name: str = Field(description="人物姓名")
    age: int = Field(description="人物年龄")
    skills: List[str] = Field(description="技能列表")

# 2. 创建解析器（自动生成 JSON Schema → 注入 prompt）
parser = PydanticOutputParser(pydantic_object=PersonInfo)

# 3. 链式调用
chain = prompt | llm | parser
result = chain.invoke({"text": "张三今年30岁，擅长Python和Go语言。"})
# → PersonInfo(name='张三', age=30, skills=['Python', 'Go语言'])
```

**内部机制**：
1. `.get_format_instructions()` → 调用 Pydantic 的 `.model_json_schema()` → 生成 JSON Schema → 嵌入预设指令模板
2. 解析时：先 `JsonOutputParser` 转 dict → 再用 `model_validate()` 验证类型 → 返回 Pydantic 实例

### 2.2 LlamaIndex 输出解析

两大组件：

- **响应合成（Response Synthesis）**：智能组合检索到的 Nodes + 用户查询，支持 `refine`（迭代优化）、`compact`（压缩调用）等模式
- **结构化输出（Pydantic Programs）**：与 LangChain 思路一致——定义 Schema → 引导生成（支持 Function Calling 则优先使用）→ 解析验证

### 2.3 不依赖框架的实现思路

通过纯提示工程实现：

| 技巧 | 说明 |
|------|------|
| 明确要求 JSON | 在 prompt 中写"只返回 JSON，不要解释" |
| 提供 JSON Schema | 描述每个字段含义和类型 |
| Few-shot 示例 | 给 1-2 组"输入→期望输出"样例 |
| 语法约束 | `llama.cpp` 的 GBNF 语法文件，强制约束 token 生成 |

## 三、Function Calling（Tool Calling）

### 3.1 工作流程

```
User → Model → Code → Tool → Model → User
```

1. **定义工具** — 用 JSON Schema 描述函数名称、描述、参数
2. **模型决策** — LLM 分析意图，返回 `tool_calls` 指令（"调某某函数，参数是 X"）
3. **代码执行** — 解析指令，实际调用 API/函数
4. **结果反馈** — 将工具结果包装为 `role: tool` 消息返回给模型
5. **最终生成** — 模型结合结果生成自然语言回答

### 3.2 实践要点

```python
# 第一次调用：User → Model
messages = [{"role": "user", "content": "杭州今天天气怎么样？"}]
message = send_messages(messages, tools=tools)

# 处理 tool_calls
if message.tool_calls:
    messages.append(message)               # 添加模型响应
    messages.append({                       # 添加工具执行结果
        "role": "tool",
        "tool_call_id": tool_call.id,
        "content": "24℃，晴朗"
    })

# 第二次调用：Tool → Model，获取最终回答
final_message = send_messages(messages, tools=tools)
```

### 3.3 优势对比

| 维度 | 纯 Prompt 工程 | Function Calling |
|------|---------------|-----------------|
| 可靠性 | 格式不稳定，易跑偏 | 模型原生支持，精确稳定 |
| 意图识别 | 被动按模板输出 | 主动选择最合适的工具 |
| 外部交互 | 纯文本生成 | 可调用 API、查 DB、控制设备 |
